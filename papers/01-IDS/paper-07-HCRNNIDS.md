## 1. 研究背景

### 1.1 问题陈述

过去 20 年，机器学习 (ML) 在网络安全领域得到广泛应用。近年来，深度学习 (DL) 因其更出色的表现逐渐取代传统方法。

### 1.2 研究动机

- **单一方法局限性**：传统 ML 或单一 DL 模型难以同时捕捉空间和时间特征
- **混合模型优势**：结合多种模型的优势可提升检测性能
- **类别不平衡**：入侵检测数据集中正常样本远多于攻击样本

---

## 2. 核心架构

### 2.1 HCRNN 模型结构

```
┌─────────────────────────────────────────────────────────────────┐
│                    HCRNNIDS 架构                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   输入数据                                                        │
│      │                                                          │
│      ▼                                                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │              CNN 层 (特征提取器)                          │   │
│   │   ├── 卷积层 → 局部空间特征提取                          │   │
│   │   └── 池化层 → 特征降维                                  │   │
│   └─────────────────────────────────────────────────────────┘   │
│      │                                                          │
│      ▼                                                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │              RNN 层 (序列建模)                           │   │
│   │   ├── 循环层 → 时间依赖关系捕捉                         │   │
│   │   └── LSTM/GRU → 长期依赖学习                           │   │
│   └─────────────────────────────────────────────────────────┘   │
│      │                                                          │
│      ▼                                                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │              分类器层                                    │   │
│   │   └── 全连接层 + Softmax → 多分类输出                   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 模块详解

#### CNN 组件

| 组件 | 功能 |
|------|------|
| 卷积层 | 提取局部空间特征 |
| 池化层 | 降维，增强特征 |
| Dropout | 防止过拟合 |

#### RNN 组件

| 组件 | 功能 |
|------|------|
| 循环层 | 捕捉时序依赖 |
| LSTM/GRU | 解决梯度消失 |
| 双向 RNN | 双向上下文信息 |

## 3. 研究方法

### 3.1 技术创新点

1. **深度+浅层混合**：结合深度模型和浅层模型优势
2. **时空特征融合**：CNN 捕捉空间 + RNN 捕捉时间
3. **类别不平衡处理**：采用重采样或代价敏感学习

### 3.2 处理流程

```
原始流量数据
    │
    ▼
数据预处理
├── 特征标准化
├── 类别编码
└── 训练/测试划分
    │
    ▼
HCRNN 模型训练
├── CNN 特征提取
├── RNN 序列建模
└── 分类输出
    │
    ▼
性能评估
├── 准确率
├── 精确率/召回率
└── F1-Score
```

---

## 4. 主要贡献

1. **混合架构设计**：首次将 CNN 和 RNN 结合用于入侵检测
2. **处理类别不平衡**：解决入侵检测数据中类别不均衡问题
3. **性能提升**：在多个指标上优于单一模型方法
4. **鲁棒性增强**：提高模型对新型攻击的检测能力

### 5.1 评估指标

|------|------|

| 模型 | 预期表现 |
|------|----------|
| 单一 CNN | 空间特征强，时间弱 |
| 单一 RNN | 时间特征强，空间弱 |
| **HCRNN** | **时空兼顾最优** |

## 6. 关键洞察

### 6.1 CNN + RNN 协同优势

```
CNN 的贡献:
├── 局部特征提取
├── 平移不变性
└── 层次化表示

RNN 的贡献:
├── 时序依赖建模
├── 变长序列处理
└── 上下文理解

结合效果: 1 + 1 > 2
```

### 6.2 类别不平衡处理策略

- 数据级：过采样/欠采样
- 算法级：类别权重调整
- 集成级：Balanced Random Forest

## 7. 与其他方法比较

| 方法 | 优点 | 缺点 |
|------|------|------|
| 传统 ML | 可解释性好 | 特征工程复杂 |
| 单一 DL | 端到端学习 | 难以捕捉时间依赖 |
| **HCRNN** | **时空特征融合** | 模型复杂 |
